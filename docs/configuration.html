Configuration

Customize C-napse to work with your preferred LLM providers, configure memory settings, and tune agent behavior to match your workflow.

Config File Location

C-napse stores configuration in your home directory:

File Structure 
~/.cnapse/
├── config.toml      # Main settings
├── credentials.toml # API keys (encrypted)
└── memory.db        # Conversation memory

Full Configuration

~/.cnapse/config.toml 
# C-napse Configuration

[llm]
provider = "ollama"          # Default: ollama, anthropic, openai, openrouter
model = "qwen2.5:0.5b"        # Default model

[llm.ollama]
base_url = "http://localhost:11434"

[llm.anthropic]
# api_key loaded from ANTHROPIC_API_KEY env var

[llm.openrouter]
site_url = "https://cnapse.app"
app_name = "C-napse"

[memory]
enabled = true
max_context_messages = 20

[agent]
max_iterations = 10
temperature = 0.7
timeout_seconds = 300

[tools]
shell_enabled = true
file_enabled = true
web_enabled = true
screenshot_enabled = true

LLM Providers

C-napse supports multiple LLM providers out of the box:

O

Ollama

Local inference

Run models locally with no API costs. Great for privacy and offline use.

base_url = "http://localhost:11434"
A

Anthropic

Claude models

Access Claude 3.5 Sonnet and other Anthropic models via their API.

ANTHROPIC_API_KEY=sk-ant-...
R

OpenRouter

Multi-provider

Access 200+ models through a single API. Great for model variety.

OPENROUTER_API_KEY=sk-or-...
G

OpenAI

GPT models

Use GPT-4o, GPT-4 Turbo, and other OpenAI models.

OPENAI_API_KEY=sk-...

Environment Variables

Set API keys as environment variables for security:

PowerShell (Windows) 
# Set permanently for current user
[Environment]::SetEnvironmentVariable("ANTHROPIC_API_KEY", "sk-ant-...", "User")
[Environment]::SetEnvironmentVariable("OPENROUTER_API_KEY", "sk-or-...", "User")
Bash (macOS / Linux) 
# Add to ~/.bashrc or ~/.zshrc
export ANTHROPIC_API_KEY="sk-ant-..."
export OPENROUTER_API_KEY="sk-or-..."
Security Note

Never commit API keys to version control. Always use environment variables instead of storing them in config files.

CLI Configuration

You can also configure C-napse via the command line:

Terminal 
# View all settings
cnapse config list

# Get a specific setting
cnapse config get llm.provider

# Set a value
cnapse config set llm.provider anthropic
cnapse config set llm.model claude-3-5-sonnet-20241022

# Set API key securely
cnapse auth set anthropic sk-ant-...
Runtime Override

Override provider at runtime: cnapse --provider anthropic "your task"
