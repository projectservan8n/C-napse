Configuration

Configure C-napse to work the way you want.

Config Files

C-napse stores configuration in your home directory:

~/.cnapse/
├── config.toml      # Main settings
├── credentials.toml # API keys (encrypted)
└── models/          # Local model storage

You can edit these files directly or use the CLI:

# View all settings
cnapse config list

# Get a specific setting
cnapse config get inference.provider

# Set a value
cnapse config set inference.provider anthropic

Inference Settings

Setting Default Description
inference.provider ollama AI provider (ollama, anthropic, openai)
inference.model qwen2.5:0.5b Model to use
inference.temperature 0.7 Response creativity (0-1)
inference.max_tokens 4096 Max response length

Provider Configuration

Ollama (Local)

# List available models
ollama list

# Pull a new model
ollama pull llama3.2:3b

# Set as default
cnapse config set inference.model llama3.2:3b

Anthropic

# Set API key
cnapse auth set anthropic sk-ant-...

# Use Claude models
cnapse config set inference.provider anthropic
cnapse config set inference.model claude-sonnet-4-20250514

OpenAI

# Set API key
cnapse auth set openai sk-...

# Use GPT models
cnapse config set inference.provider openai
cnapse config set inference.model gpt-4o

Telegram Bot

Control your PC remotely via Telegram:

# Set bot token (from @BotFather)
cnapse auth set telegram YOUR_BOT_TOKEN

# Allow specific users (optional)
cnapse config set telegram.allowed_users [123456789]

# Start the bot
cnapse telegram
Security: Always restrict allowed users in production. Without restrictions, anyone can control your PC.
