Agent System

Understanding C-napse's multi-agent architecture.

Overview

C-napse uses a multi-agent system where specialized agents handle different types of tasks. A router agent analyzes your query and delegates to the most appropriate specialist. 

┌─────────────────────────────────────────┐
│              User Query                 │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│            Router Agent                 │
│   Classifies intent, selects agent      │
└──────────────────┬──────────────────────┘
                   │
       ┌───────────┼───────────┐
       ▼           ▼           ▼
   ┌───────┐  ┌────────┐  ┌──────────┐
   │ Shell │  │  Code  │  │ Research │
   │ Agent │  │ Agent  │  │  Agent   │
   └───────┘  └────────┘  └──────────┘

Router Agent

The router is the brain of the system. It analyzes queries to determine:

  • Intent classification — What does the user want?
  • Agent selection — Which specialist should handle this?
  • Context extraction — What information is needed?

You can force a specific agent:

cnapse -a shell "list all rust files"
cnapse -a code "explain this function"

Shell Agent

Handles system operations, file management, and command execution.

Capabilities

  • Execute shell commands
  • File and directory operations
  • Process management
  • System information queries

Example Queries

"list all files in this folder"
"find large files over 100MB"
"show running processes"
"what's my disk usage?"
"rename all .txt files to .md"

Code Agent

Specializes in code understanding, generation, and modification.

Capabilities

  • Code explanation and analysis
  • Bug identification
  • Refactoring suggestions
  • Code generation

Example Queries

"explain what this function does"
"find potential bugs in this file"
"refactor this code to be more readable"
"write a test for this function"

Research Agent

Handles information gathering and knowledge synthesis.

Capabilities

  • Answer general questions
  • Explain concepts
  • Compare technologies
  • Summarize documentation

Example Queries

"what's the difference between TCP and UDP?"
"explain async/await in Rust"
"how does memory management work in Go?"
"compare React vs Vue for this use case"
