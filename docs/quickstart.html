Quick Start

Get C-napse running on your machine in under 2 minutes.

Installation

Install C-napse with a single command:

irm https://raw.githubusercontent.com/projectservan8n/C-napse/main/install.ps1 | iex
curl -fsSL https://raw.githubusercontent.com/projectservan8n/C-napse/main/install.sh | bash

This will:

  • Download the latest pre-built binary
  • Add it to your PATH
  • Install Ollama (if not present)
  • Pull a lightweight AI model

First Run

After installation, restart your terminal and run:

cnapse

This launches the interactive TUI (Terminal User Interface), similar to Claude Code.

Tip: You can also use the shorter aliases cn or c.

Setup Providers

C-napse works with multiple AI providers. By default, it uses Ollama for local inference.

Local (Ollama)

Already configured during installation. To use a different model:

cnapse config set inference.model qwen2.5:7b

Cloud Providers

For better quality responses, configure a cloud provider:

# Anthropic (Claude)
cnapse auth set anthropic YOUR_API_KEY

# OpenAI
cnapse auth set openai YOUR_API_KEY

# Set as default
cnapse config set inference.provider anthropic

Commands

Command Description
cnapse Start interactive TUI
cnapse "query" Run a single query
cnapse run Interactive mode with options
cnapse serve Start API server
cnapse config Manage configuration
cnapse auth Manage API keys

TUI Mode

The TUI provides a Claude Code-like experience:

  • Type your query and press Enter
  • Ctrl+C to exit
  • Ctrl+L to clear screen
  • Up/Down arrows for history

Query Mode

For quick one-off queries:

# Ask a question
cnapse "what files are in this folder?"

# With specific provider
cnapse -p anthropic "explain this code"

# With specific agent
cnapse -a code "refactor this function"
