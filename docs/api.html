API Reference

C-napse includes a server mode that exposes both REST and WebSocket APIs for integration with other applications.

Starting the Server

Terminal 
# Start the server with default settings
cnapse serve

# With custom port
cnapse serve --port 8080

# Without web UI
cnapse serve --no-web

Default URL: http://localhost:3000

REST API

Health Check

GET /api/health

Check if the server is running.

Response 
{
  "status": "ok",
  "version": "0.8.0"
}

Chat

POST /api/chat

Send a message and get a response from C-napse.

Request 
{
  "message": "list files in current directory",
  "provider": "ollama",  // optional
  "agent": "shell",      // optional
  "stream": false        // optional
}
Response 
{
  "id": "msg_123",
  "content": "Here are the files...",
  "agent": "shell",
  "tokens": 150,
  "tools_used": ["list_dir"]
}

List Agents

GET /api/agents

Get a list of available agents.

Response 
{
  "agents": [
    {"name": "computer", "description": "Desktop automation"},
    {"name": "browser", "description": "Web automation"},
    {"name": "vision", "description": "Screen analysis"},
    {"name": "shell", "description": "System operations"},
    {"name": "adaptive", "description": "Self-correcting AI"}
  ]
}

Chat History

GET /api/chat/history

Retrieve conversation history.

DELETE /api/chat/history

Clear conversation history.

WebSocket API

For real-time streaming responses, connect via WebSocket:

Connection 
ws://localhost:3000/ws?token=your-api-key

Sending Messages

Client message 
{
  "type": "chat",
  "id": "msg_123",
  "content": "explain this code",
  "stream": true
}

Receiving Responses

Streaming chunk 
{
  "type": "chunk",
  "id": "msg_123",
  "content": "This function...",
  "done": false
}
Final response 
{
  "type": "response",
  "id": "msg_123",
  "content": "Full response here",
  "agent": "code",
  "tokens": 250,
  "done": true
}

Message Types

Type Direction Description
auth Client -> Server Authenticate connection
chat Client -> Server Send a message
cancel Client -> Server Cancel current request
chunk Server -> Client Streaming response chunk
response Server -> Client Complete response
tool Server -> Client Tool execution update
error Server -> Client Error message

Authentication

Generate an API key for server access:

Terminal 
# Generate new key
cnapse auth generate-server-key

# Output: cnapse-abc123...

Include the key in requests:

HTTP Header 
X-API-Key: cnapse-abc123...
WebSocket Query Param 
ws://localhost:3000/ws?token=cnapse-abc123...
Development Mode

If no API key is configured, the server accepts all requests. Always set an API key in production.
